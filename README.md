# Python Algorithms and Tests

This repository contains a collection of Python tests and algorithms for practice and study purposes.

## Features

The repository includes:

`algorithms/`: A directory containing various Python algorithms for practice and study purposes.

### 1. `text_to_image.py`
A Python script that converts a text message into an image file. The text is mapped directly into pixel colors (RGB), where each pixel represents up to 3 ASCII characters. An additional white pixel marks the end of the message.

**Estudo de Caso (Case Study):**
For example, calling the function with the message `"Eu Te Amo"` generates an image. Let's trace it:
- `'E'` (69), `'u'` (117), `' '` (32) $\rightarrow$ Pixel 1: `RGB(69, 117, 32)`
- `'T'` (84), `'e'` (101), `' '` (32) $\rightarrow$ Pixel 2: `RGB(84, 101, 32)`
- `'A'` (65), `'m'` (109), `'o'` (111) $\rightarrow$ Pixel 3: `RGB(65, 109, 111)`
- End of Message $\rightarrow$ Pixel 4: `RGB(255, 255, 255)`
The script calculates the total pixels needed, maps these into an image of max width 500, and saves it. This is a basic form of Steganography or visual data encoding.

### 2. `image_to_text.py`
A Python script that reads an image file generated by `text_to_image.py` and converts it back to the original text message.

**Estudo de Caso (Case Study):**
When parsing the image generated in the previous step:
- It reads Pixel 1 `RGB(69, 117, 32)` and maps back to ASCII values `[69, 117, 32]` which translates to `'E', 'u', ' '`.
- It continues this process until it encounters the white pixel `RGB(255, 255, 255)`, which signals the end of the message. It correctly removes any padding character values and restores the exact text message `"Eu Te Amo"`.

## Suggestions for Improvements (Melhorias Sugeridas)

1. **Support for UTF-8 Encoding**: Currently, the algorithm relies heavily on ASCII encoding (0-255 limit). Updating the script to support UTF-8 (e.g., handling variable byte lengths) would allow encoding special characters, emojis, and characters from different alphabets.
2. **LSB Steganography (Least Significant Bit)**: Instead of explicitly replacing the image's pixels with ASCII bytes, the algorithm could be adapted to hide the text message in the least significant bits of an existing cover image. This would make the hidden message visually unnoticeable.
3. **Use of the Alpha Channel**: The algorithm uses RGB channels (3 bytes per pixel). Utilizing RGBA could allow storing 4 characters per pixel or using the Alpha channel to specify metadata like message length or encoding type.
4. **Data Compression**: Applying compression (like Gzip or Huffman) to the text before transforming it into an image can allow for drastically larger amounts of text to be stored in the same number of pixels.
5. **Encryption**: Applying AES or another encryption mechanism to the text before encoding would add a layer of security, making it impossible to retrieve the text without a key.

Here is a generous list of algorithms that would be highly beneficial to add to this repository:

### Cryptography and Hashing
* **AES Encryption/Decryption:** Block cipher implementation or wrapper for secure data encryption.
* **Base64 Encoding/Decoding:** Algorithm to convert binary data into a string format and vice versa.
* **SHA-256 Hashing:** Implementation of the secure hash algorithm to verify data integrity.
* **RSA Asymmetric Encryption:** Basic implementation of public/private key encryption.

### Compression
* **Huffman Coding:** A lossless data compression algorithm.
* **Run-Length Encoding (RLE):** A simple form of data compression where sequences of the same data value are stored as a single data value and count.
* **LZW (Lempel-Ziv-Welch) Compression:** Dictionary-based data compression technique.

### Data Structures and Graphs
* **Dijkstra's Algorithm:** Finding the shortest paths between nodes in a graph.
* **A* Search Algorithm:** Pathfinding algorithm widely used in pathfinding and graph traversal.
* **Binary Search Tree (BST) Operations:** Insertion, deletion, and traversal algorithms.

### Sorting and Searching
* **QuickSort and MergeSort:** Efficient sorting algorithms for generic data handling.
* **Binary Search:** O(log n) time complexity search algorithm on sorted structures.
